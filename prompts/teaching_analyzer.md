# 教学能力分析模板

## 分析目标

从教学材料中提取教师的教学能力，包括：
1. 学科知识体系
2. 教学方法库
3. 备课流程
4. 评价策略
5. 资源建设

## 输入数据

### 数据来源
- 教案/教学设计
- 课件（PPT）
- 学案/作业设计
- 试卷/评价材料
- 教学反思/论文
- 课堂实录文字稿

### 预处理
1. 文本清洗（去除格式、标准化）
2. 分段处理（按教学环节）
3. 关键信息提取（目标、内容、方法、评价）

## 分析维度

### 维度一：学科知识体系
**分析内容**：
- 知识结构把握
- 重点难点处理
- 知识关联设计
- 学科思想渗透

**提取指标**：
```yaml
knowledge_structure:
  scope: 知识覆盖面
  depth: 知识深度
  connections: 知识关联度
  sequencing: 知识序列设计
```

### 维度二：教学方法库
**分析内容**：
- 常用教学方法
- 方法组合模式
- 方法适用情境
- 方法创新点

**方法分类**：
- 讲授法
- 讨论法
- 探究法
- 实验法
- 案例法
- 项目学习
- 合作学习
- 翻转课堂

**提取指标**：
```yaml
methods:
  primary: 主要方法
  secondary: 辅助方法
  combinations: 方法组合模式
  adaptations: 方法适应性调整
```

### 维度三：备课流程
**分析内容**：
- 备课环节设计
- 资源准备习惯
- 时间分配模式
- 备选方案设计

**环节分析**：
1. 目标设定
2. 学情分析
3. 内容选择
4. 方法设计
5. 评价设计
6. 反思调整

### 维度四：评价策略
**分析内容**：
- 评价方式多样性
- 评价时机把握
- 反馈方式
- 评价标准设计

**评价类型**：
- 形成性评价
- 总结性评价
- 诊断性评价
- 发展性评价

### 维度五：资源建设
**分析内容**：
- 资源类型偏好
- 资源原创程度
- 资源整合能力
- 资源更新频率

**资源类型**：
- 文本资源
- 多媒体资源
- 实验器材
- 数字化资源
- 社区资源

## 分析流程

### 步骤1：材料分类
```python
# 伪代码
def classify_materials(materials):
    categories = {
        "lesson_plans": [],      # 教案
        "courseware": [],        # 课件
        "assessments": [],       # 评价材料
        "reflections": [],       # 反思记录
        "research": []           # 研究论文
    }
    # 分类逻辑
    return categories
```

### 步骤2：特征提取
对每类材料提取特征：
- **教案**：目标设计、环节安排、时间分配
- **课件**：视觉设计、信息组织、互动设计
- **评价材料**：题型分布、难度梯度、评分标准

### 步骤3：模式识别
识别教学中的固定模式：
- 课堂开场模式
- 新知引入模式
- 练习设计模式
- 课堂小结模式
- 作业布置模式

### 步骤4：能力建模
构建教学能力模型：
```yaml
teaching_competencies:
  knowledge_mastery:  # 知识掌握度
    score: 0.9
    evidence: ["教案中知识结构清晰", "重点难点处理得当"]
    
  method_application:  # 方法应用
    score: 0.85
    evidence: ["多种方法组合使用", "方法适应学情"]
    
  assessment_design:  # 评价设计
    score: 0.8
    evidence: ["评价方式多样", "及时反馈"]
    
  resource_development:  # 资源开发
    score: 0.75
    evidence: ["原创课件", "整合多方资源"]
```

## 质量评估

### 评估指标
1. **完整性**：分析覆盖的维度是否全面
2. **准确性**：提取的特征是否准确反映材料
3. **深度**：分析是否触及教学本质
4. **实用性**：分析结果是否对Skill生成有用
