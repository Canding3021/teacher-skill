# 对话纠正处理模板 v2 - 影响权重版

## 🎯 核心指令

**理解纠正意图，应用适当的影响权重，避免过度修正。**
**在保留原有建模的基础上进行微调，而不是推倒重来。**

## 📋 输出契约

```typescript
interface CorrectionResult {
  updated_skill: SkillDocument;      // 更新后的Skill文档
  correction_analysis: {
    correction_type: "fact_error" | "style_adjustment" | "context_supplement" | "principle_refinement";
    confidence: number;              // 0-1，纠正置信度
    influence_weight: number;        // 0-1，影响权重（0.1轻微，0.5中等，0.9重大）
    evidence_quality: "strong" | "medium" | "weak";
  };
  changes_applied: Change[];
  unchanged_sections: string[];      // 未修改的部分（保持原样）
  metadata: {
    correction_id: string;
    processed_at: string;
    user_feedback: string;           // 原始用户反馈
    model_interpretation: string;    // AI对反馈的理解
  };
}

interface SkillDocument {
  teaching: string;                  // 教学能力文档
  teacher: string;                   // 教师风格文档
  meta: Record<string, any>;         // 元数据
}

interface Change {
  section: string;                   // 修改的章节
  change_type: "replace" | "add" | "remove" | "modify";
  before: string;                    // 修改前内容
  after: string;                     // 修改后内容
  weight: number;                    // 0-1，这个修改的影响权重
  rationale: string;                 // 修改理由
}
```

## ⚖️ 影响权重系统

### 权重级别定义

#### 低权重 (0.1-0.3)：轻微调整
**适用场景**：
- 语言表达的微调
- 细节的补充说明
- 情境的澄清

**处理方式**：
- 最小化修改
- 保持原有结构
- 添加注释说明

**示例**：
```
用户："他不会说'我们来思考一下'，他通常说'大家动动脑子'"

影响权重：0.2
处理：仅修改口头禅列表，其他描述不变
```

#### 中权重 (0.4-0.6)：适度调整
**适用场景**：
- 行为模式的修正
- 方法的补充
- 风格的微调

**处理方式**：
- 局部修改
- 保持核心不变
- 标注调整原因

**示例**：
```
用户："他不会公开批评学生，但会在私下严肃指出问题"

影响权重：0.5  
处理：修改反馈模式描述，从"温和"调整为"私下严肃"
```

#### 高权重 (0.7-0.9)：重大调整
**适用场景**：
- 核心理念的修正
- 重大事实错误
- 原则性调整

**处理方式**：
- 结构性修改
- 可能影响多个部分
- 需要强证据支持

**示例**：
```
用户："他根本不是'以学生为中心'，他是典型的'知识本位'"

影响权重：0.8
处理：重新分析教育理念，调整Layer 0描述
```

## 🔍 纠正意图分析

### 意图分类矩阵

| 用户表达 | 可能意图 | 建议权重 | 处理策略 |
|---------|----------|----------|----------|
| "他不会这样说" | 语言修正 | 0.2 | 修改具体表述 |
| "这不对，他应该是..." | 行为修正 | 0.5 | 调整行为描述 |
| "他还会..." | 补充说明 | 0.3 | 添加补充内容 |
| "在...情况下，他会..." | 情境澄清 | 0.4 | 添加情境说明 |
| "这要看情况..." | 复杂性补充 | 0.6 | 增加条件描述 |
| "他根本不是..." | 原则修正 | 0.8 | 重大调整 |

### 置信度评估

#### 高置信度 (>0.7)
- 用户提供具体例子
- 纠正内容明确具体
- 与已知信息一致
- 有多个证据支持

#### 中置信度 (0.4-0.7)
- 纠正内容合理但缺乏细节
- 单次提及，无例子
- 与部分信息一致

#### 低置信度 (<0.4)
- 模糊表述
- 主观判断
- 与现有证据矛盾
- 需要进一步澄清

## 🛠️ 纠正处理流程

### 步骤1：意图解析
1. **提取纠正点**：
   - 什么需要修改？（具体内容）
   - 应该改成什么？（目标状态）
   - 为什么需要改？（原因）

2. **分析纠正深度**：
   - 表面层：具体话语、行为
   - 模式层：行为模式、习惯
   - 原则层：核心理念、价值观

### 步骤2：权重分配
基于以下因素分配权重：
1. **证据强度**：用户提供的证据质量
2. **影响范围**：纠正影响的范围大小
3. **一致性**：与现有信息的一致性
4. **重要性**：对Skill准确性的重要性

### 步骤3：实施纠正
根据权重选择实施策略：

#### 策略A：最小化修改（权重<0.3）
- 仅修改具体表述
- 保持其他部分不变
- 添加轻微调整说明

#### 策略B：局部优化（权重0.3-0.6）
- 修改相关部分
- 调整描述方式
- 保持核心结构

#### 策略C：结构性调整（权重>0.6）
- 重新分析相关部分
- 可能影响多个章节
- 需要更谨慎处理

### 步骤4：验证与反馈
1. **内部一致性检查**：修改后是否自洽
2. **证据匹配检查**：修改是否有证据支持
3. **用户确认**：向用户展示修改内容

## 📊 纠正质量保证

### 防过度修正机制
1. **变更影响分析**：评估每个修改的影响范围
2. **历史版本对比**：与之前版本对比变化程度
3. **用户意图验证**：确认理解正确

### 保留原有价值
1. **识别保留内容**：明确哪些部分应该保持
2. **维护核心结构**：保持Skill的基本框架
3. **保护已验证信息**：保护已有强证据支持的信息

### 渐进式修正
1. **小步快跑**：多次小修正优于一次大修正
2. **用户验证**：每次修正后获取用户反馈
3. **累积学习**：从多次修正中学习模式


## 🎯 特殊情况处理

### 模糊纠正
用户："感觉不太像他"
处理：
1. 请求具体例子
2. 如果无法提供，应用最低权重调整
3. 标注为"需要进一步澄清"

### 矛盾纠正
不同用户提供矛盾纠正：
```
用户A："他很严肃"
用户B："他很幽默"
```
处理：
1. 分析可能的情境差异
2. 创建情境分层描述
3. 标注存在不同视角

### 渐进式发展
多次纠正体现发展变化：
```
3月：新手期，模仿学习
6月：形成个人风格
9月：创新教学方法
```
处理：
1. 按时间线组织
2. 标注发展阶段
3. 展示成长轨迹
