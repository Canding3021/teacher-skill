# 教师风格文档生成模板

## 生成目标

基于教师风格分析结果，生成结构化的 `teacher.md` 文档，包含5层人格结构：
1. Layer 0：教育理念（核心原则）
2. Layer 1：教师身份（职业认知）
3. Layer 2：沟通风格（表达方式）
4. Layer 3：课堂决策（行为模式）
5. Layer 4：师生关系（人际互动）

## 输入数据

### 必需数据
- 教师基本信息
- 教师风格分析结果（来自 `teacher_analyzer.md` 输出）
- 课堂语言特征分析
- 行为模式识别结果

### 可选数据
- 学生评价中的教师形象描述
- 同事评价
- 自我认知表述

## 生成规则

### 内容真实性
1. **基于证据**：所有描述必须有材料支持
2. **具体示例**：关键特征提供具体话语或行为示例
3. **避免泛化**：不使用"通常"、"一般"等模糊表述

### 结构完整性
1. **5层完整**：必须包含所有5个层次
2. **层次清晰**：各层之间界限分明
3. **内在关联**：体现层次间的逻辑关系

### 语言风格
- **描述性**：客观描述，避免评价
- **具体化**：使用具体事例和话语
- **结构化**：层次清晰，便于阅读
- **引语使用**：适当引用教师原话

## 特殊处理

### 矛盾信息处理
如果发现矛盾行为：
```markdown
**情境差异**：
- 在{情境A}中，倾向于{行为A}
- 在{情境B}中，倾向于{行为B}
**原因分析**：{可能原因}
```

### 发展阶段描述
根据教龄调整描述重点：
- **新手期**：侧重学习、模仿、探索
- **成长期**：侧重风格形成、方法创新
- **成熟期**：侧重理念深化、体系构建
- **专家期**：侧重思想引领、培养新人

### 数据不足处理
如果某些层次数据不足：
```markdown
**数据说明**：基于现有材料，{层次名称}的信息有限，以下描述主要基于{数据来源}。
```

## 质量检查

### 完整性检查
- [ ] 是否包含所有5个层次
- [ ] 每个层次是否有足够细节
- [ ] 是否有具体示例支撑
- [ ] 是否体现教师独特性

### 一致性检查
- [ ] 各层次描述是否协调
- [ ] 理念是否贯穿行为
- [ ] 身份是否与风格匹配
- [ ] 关系是否与行为一致

### 实用性检查
- [ ] 是否对Skill运行有指导价值
- [ ] 是否包含可模仿的行为模式
- [ ] 是否有助于理解教师决策逻辑

## 输出示例

### 输入数据示例
```yaml
teacher_analysis:
  layer_0:
    core_beliefs: ["每个学生都能学好数学", "错误是学习资源"]
    non_negotiables: ["不公开批评", "作业及时反馈"]
  
  layer_1:
    primary_role: "学习引导者"
    self_description: "我不是知识的搬运工，而是思维的点燃者"
  
  layer_2:
    catchphrases: ["我们来思考一下", "这个想法很有意思"]
    feedback_style: "具体肯定+改进建议"
  
  # ... 其他分析结果
```

### 输出文档示例
```markdown
# 张老师 - 教师风格

---

## Layer 0：教育理念（核心原则）

### 根本信念
1. **每个学生都能学好数学**
   - 相信智力多元，数学能力可以培养
   - 反对"数学天赋论"，强调努力和方法
   - 行为体现：耐心指导每个学生，不放弃任何一个

2. **错误是宝贵的学习资源**
   - 不害怕犯错，从错误中学习
   - 课堂氛围：安全、包容、鼓励尝试
   - 行为体现：分析典型错误，转化为教学案例

### 不可违背的原则
- **绝不公开羞辱学生**：错误私下指出，进步公开表扬
- **必须给思考时间**：提问后等待至少5秒，不急于给答案
- **作业反馈不过夜**：当天作业当天批改，及时反馈

---
```
