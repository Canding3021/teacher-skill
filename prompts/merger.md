# 增量合并模板

## 功能说明

当用户提供新的教学材料时，分析增量内容并合并到现有Skill中。

## 输入数据

### 现有Skill数据
- 当前 teaching.md 内容
- 当前 teacher.md 内容
- 当前 meta.json 数据

### 新增材料
- 新教学材料（教案、课件、反思等）
- 新课堂记录
- 新学生评价
- 用户补充描述

## 分析流程

### 步骤1：材料分类与解析
1. **文件类型识别**
   - 教案/教学设计
   - 课件/PPT
   - 课堂实录/录像文字稿
   - 教学反思/日记
   - 学生作业/评价
   - 教研记录/论文

2. **内容解析**
   - 提取文本内容
   - 识别关键信息
   - 结构化处理

### 步骤2：增量特征提取
1. **教学能力增量**
   - 新教学方法
   - 新课堂环节设计
   - 新评价策略
   - 新资源类型

2. **教师风格增量**
   - 新口头禅/高频词
   - 新行为模式
   - 新决策案例
   - 新关系互动

### 步骤3：与现有内容对比
1. **一致性检查**
   - 新增内容是否与现有特征一致
   - 是否存在矛盾或变化
   - 是补充还是修正

2. **重要性评估**
   - 出现频率
   - 独特性程度
   - 教学价值
   - 代表性强度

### 步骤4：合并策略选择
根据增量内容类型选择合并策略：

#### 策略A：补充添加（新增内容）
- **适用情况**：与现有内容不冲突，是新的补充
- **处理方式**：追加到相应章节
- **示例**：新的教学案例、新的口头禅

#### 策略B：深化细化（细节补充）
- **适用情况**：对现有内容的细节补充
- **处理方式**：在现有描述中添加细节
- **示例**：现有方法的更具体实施步骤

#### 策略C：修正更新（矛盾内容）
- **适用情况**：与现有内容矛盾，需要修正
- **处理方式**：更新现有描述，保留历史版本
- **示例**：教学风格的变化、方法的改进

#### 策略D：整合重构（重大变化）
- **适用情况**：重大变化，需要结构调整
- **处理方式**：部分重构，重新组织内容
- **示例**：教学理念的重大转变

## 合并规则

### 教学能力合并规则
1. **知识体系**
   - 新知识点：添加到知识结构
   - 新关联：补充到知识关联描述
   - 新重点：更新重点难点列表

2. **教学方法**
   - 新方法：添加到方法库
   - 方法改进：更新方法描述
   - 方法组合：补充组合模式

3. **备课流程**
   - 新环节：添加到备课步骤
   - 时间调整：更新时间分配
   - 流程优化：更新流程描述

4. **评价策略**
   - 新评价方式：添加到评价方式
   - 反馈改进：更新反馈特点
   - 标准调整：更新评价标准

### 教师风格合并规则
1. **教育理念**
   - 新理念：谨慎添加，需多次出现
   - 理念深化：补充具体体现
   - 理念调整：标注发展阶段

2. **沟通风格**
   - 新口头禅：添加到口头禅列表
   - 风格变化：标注情境差异
   - 情感调整：更新情感基调描述

3. **行为模式**
   - 新模式：添加到相应部分
   - 模式优化：更新模式描述
   - 情境应对：补充情境说明

4. **师生关系**
   - 新互动：添加到关系描述
   - 关系变化：标注变化原因
   - 支持扩展：补充支持方式

## 冲突处理

### 冲突类型
1. **直接矛盾**
   - 现有："作业当天批改"
   - 新增："作业隔天批改"
   - 处理：分析原因，可能是发展阶段变化

2. **情境差异**
   - 现有：公开课严谨规范
   - 新增：常态课轻松随意
   - 处理：补充情境说明

3. **发展阶段**
   - 现有：新手期模仿学习
   - 新增：成熟期创新教学
   - 处理：标注发展阶段

### 冲突解决流程
1. **识别冲突**：明确冲突点
2. **分析原因**：情境、阶段、材料类型
3. **选择策略**：修正、补充说明、版本管理
4. **实施合并**：按策略处理
5. **记录说明**：在文档中添加说明

## 版本管理

### 版本创建
每次合并前创建版本存档：
```bash
python3 tools/version_manager.py --action backup --slug {slug}
```

### 变更记录
在文档中添加变更记录：
```markdown
## 更新记录

### 版本 {new_version} ({date})
- 新增：{新增内容摘要}
- 更新：{更新内容摘要}
- 修正：{修正内容摘要}
```

### 回滚支持
保留历史版本，支持回滚：
```bash
python3 tools/version_manager.py --action rollback --slug {slug} --version {old_version}
```

## 质量检查

### 合并后检查
1. **完整性**
   - 新增内容是否完整合并
   - 原有内容是否被错误修改
   - 结构是否保持清晰

2. **一致性**
   - 新增与原有内容是否协调
   - 各部分描述是否一致
   - 语言风格是否统一

3. **准确性**
   - 合并是否准确反映新增内容
   - 是否有误解或遗漏
   - 是否保持教师真实特点

### 用户确认
向用户展示合并摘要：
```
本次合并摘要：

教学能力更新：
  - 新增教学方法：{method}
  - 补充备课环节：{step}
  - 更新评价策略：{strategy}

教师风格更新：
  - 新增口头禅："{catchphrase}"
  - 补充行为模式：{pattern}
  - 深化教育理念：{philosophy}

确认合并？如需调整请告诉我。
```

## 输出处理

### 文件更新
1. **teaching.md**：按规则更新教学能力内容
2. **teacher.md**：按规则更新教师风格内容
3. **meta.json**：更新版本号和更新时间
4. **SKILL.md**：重新生成完整Skill文件

### 变更摘要
生成变更摘要文件：
```json
{
  "slug": "zhangsan",
  "version_from": "v2",
  "version_to": "v3",
  "merged_at": "2026-04-10T12:00:00Z",
  "changes": {
    "teaching": {
      "added": [...],
      "updated": [...],
      "removed": [...]
    },
    "teacher": {
      "added": [...],
      "updated": [...],
      "removed": [...]
    }
  },
  "source_materials": ["file1.docx", "file2.pdf"]
}
```

## 特殊情况处理

### 大量新增材料
如果一次性添加大量材料：
1. **分批处理**：按材料类型分批分析
2. **优先级排序**：重要材料优先处理
3. **渐进合并**：多次小合并代替一次大合并

### 质量较差材料
如果材料质量较差（模糊、不完整）：
1. **质量评估**：标注材料质量
2. **选择性合并**：只合并可靠部分
3. **请求补充**：提示用户提供更好材料

### 时间跨度材料
如果材料时间跨度大：
1. **时间标注**：标注材料时间
2. **发展阶段**：按时间分析发展阶段
3. **演变描述**：描述风格演变过程

## 最佳实践

### 合并频率
- **建议**：积累一定量新材料后合并
- **避免**：每个小文件都单独合并
- **平衡**：及时性与合并质量平衡

### 材料准备
- **建议**：提供高质量、代表性材料
- **避免**：提供大量重复、低质材料
- **提示**：指导用户准备有效材料

### 用户参与
- **建议**：重要变更请用户确认
- **避免**：完全自动合并可能的重要变化
- **协作**：与用户协作完成高质量合并
