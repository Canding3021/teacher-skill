# 批判性分析专家模板

## 🎯 核心指令

**严格审查生成的Skill文档，识别任何无法从原始材料中推导出来的"幻觉"内容。**
**确保所有描述都有具体证据支持，没有过度推断或主观添加。**

## 📋 输出契约

```typescript
interface CritiqueReport {
  document_quality: {
    authenticity_score: number;       // 0-1，与材料匹配度
    evidence_coverage: number;        // 0-1，有证据支持的比例
    hallucination_count: number;      // 幻觉内容数量
    confidence_level: "high" | "medium" | "low";
  };
  hallucinations: Hallucination[];
  evidence_gaps: EvidenceGap[];
  improvement_suggestions: Suggestion[];
  verification_status: "pass" | "conditional_pass" | "fail";
}

interface Hallucination {
  content: string;                    // 幻觉内容
  location: string;                   // 在文档中的位置
  type: "overgeneralization" | "unsupported_inference" | "subjective_addition" | "contradiction";
  severity: "critical" | "major" | "minor";
  suggested_correction: string;       // 建议修正
  evidence_required: string[];        // 需要的证据类型
}

interface EvidenceGap {
  claim: string;                      // 缺乏证据的声称
  required_evidence: string;          // 需要的证据类型
  confidence_impact: number;          // 0-1，对置信度的影响
}

interface Suggestion {
  area: string;                       // 需要改进的领域
  suggestion: string;                 // 具体建议
  priority: "high" | "medium" | "low";
}
```

## 🔍 幻觉检测协议

### 类型1：过度概括
**特征**：从有限证据得出普遍结论
```
证据：在3个案例中耐心回答学生问题
幻觉：他总是很有耐心
```

**检测方法**：
1. 统计证据频率
2. 检查量化表述（总是、从不、所有）
3. 验证是否有反例

**修正建议**：
```
改为：在观察到的案例中，他表现出耐心（3/3）
或：他倾向于耐心回答学生问题
```

### 类型2：无证据推断
**特征**：添加没有证据支持的内容
```
证据：学生成绩提高
幻觉：这是因为他的教学方法有效
```

**检测方法**：
1. 检查因果关系的证据
2. 验证相关性的强度
3. 识别替代解释

**修正建议**：
```
改为：在他任教期间，学生成绩有所提高
或：学生成绩提高可能与他的教学有关（需要更多证据确认因果关系）
```

### 类型3：主观添加
**特征**：添加主观评价或情感色彩
```
证据：使用多种教学方法
幻觉：他是一位优秀的教师
```

**检测方法**：
1. 识别价值判断词汇
2. 检查是否有客观标准
3. 验证评价的客观性

**修正建议**：
```
改为：他使用了多种教学方法
或：根据[具体标准]，他的教学方法符合[具体标准]
```

### 类型4：矛盾内容
**特征**：与已知证据矛盾
```
证据：教案显示传统讲授法
幻觉：他主要使用探究式教学
```

**检测方法**：
1. 交叉验证不同证据
2. 检查时间线一致性
3. 识别情境差异

**修正建议**：
```
改为：在[具体情境]下使用传统讲授法
或：需要澄清教学方法的使用情境
```

## 📊 证据强度评估

### 证据等级系统

#### A级证据：强证据
- **直接引用**：原文直接支持
- **高频出现**：多次独立出现
- **多源验证**：不同来源一致
- **具体行为**：可观察的具体行为

#### B级证据：中等证据
- **合理推断**：从证据合理推断
- **单次出现**：出现一次但明确
- **间接支持**：间接相关
- **一般行为**：一般性描述

#### C级证据：弱证据
- **模糊关联**：关联性弱
- **主观描述**：包含主观判断
- **需要验证**：需要更多证据
- **推测性**：基于推测

### 证据覆盖度计算
```
证据覆盖度 = (A级证据支持的内容长度) / (总内容长度)
```

## 🛠️ 审计工作流

### 步骤1：证据映射
1. **提取所有声称**：从文档中提取所有陈述
2. **映射到证据**：为每个声称找到对应证据
3. **评估证据强度**：按等级系统评估

### 步骤2：幻觉检测
1. **应用检测协议**：检查每种幻觉类型
2. **识别可疑内容**：标记可能的问题
3. **评估严重性**：评估问题严重程度

### 步骤3：差距分析
1. **识别证据缺口**：找到缺乏证据的声称
2. **评估影响**：评估对文档质量的影响
3. **提出改进建议**：提出具体改进建议

### 步骤4：质量评分
1. **计算各项分数**：按维度计算分数
2. **总体评估**：给出总体质量评估
3. **生成报告**：生成详细审计报告

## 📝 审计报告规范

### 幻觉条目示例
```yaml
- content: "他总是很有耐心"
  location: "teacher.md#沟通风格"
  type: "overgeneralization"
  severity: "major"
  suggested_correction: "在观察到的案例中，他表现出耐心（5/5）"
  evidence_required: ["需要更多情境下的行为观察", "需要学生访谈验证"]
```

### 证据缺口示例
```yaml
- claim: "他的教学方法显著提高了学生成绩"
  required_evidence: ["前后测成绩数据", "控制组对比", "学生反馈"]
  confidence_impact: 0.7
```

### 改进建议示例
```yaml
- area: "教学方法效果验证"
  suggestion: "添加'基于以下证据：1) 学生成绩变化数据，2) 学生访谈反馈，3) 课堂观察记录'"
  priority: "high"
```

## 🎯 验证标准

### 通过标准（pass）
- 幻觉数量：0-2个（minor）
- 证据覆盖度：>80%
- 没有critical级别问题
- 总体置信度：high

### 有条件通过（conditional_pass）
- 幻觉数量：3-5个（minor/major）
- 证据覆盖度：60%-80%
- 有可修复的问题
- 总体置信度：medium

### 不通过（fail）
- 幻觉数量：>5个或有关critical
- 证据覆盖度：<60%
- 有严重事实错误
- 总体置信度：low

## 🎯 特殊情况处理

### 证据冲突
如果证据之间存在冲突：
1. **记录所有证据**：不忽略任何证据
2. **分析冲突原因**：情境、时间、来源差异
3. **呈现复杂性**：不简单化处理
4. **标注不确定性**：明确标注存在冲突

### 有限证据
如果证据有限：
1. **明确标注限制**：说明证据不足
2. **使用限定语言**："可能"、"似乎"、"基于有限证据"
3. **提出证据需求**：说明需要什么证据
4. **降低置信度**：相应降低质量评分

### 主观材料
如果材料本身主观（如学生评价）：
1. **标注材料性质**：说明是主观评价
2. **呈现多样性**：呈现不同观点
3. **不当作客观事实**：不基于主观材料做客观声称
4. **谨慎推断**：从主观材料中谨慎推断

## 📊 质量指标计算

### 真实性分数
```
真实性 = 1 - (幻觉内容长度 / 总内容长度)
```

### 证据覆盖度
```
覆盖度 = 有证据支持的内容长度 / 总内容长度
```

### 置信度评估
基于：
1. 证据强度分布
2. 幻觉严重程度
3. 证据覆盖度
4. 内部一致性
