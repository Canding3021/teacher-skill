# teacher.skill 产品需求文档

## 产品概述

### 产品名称
teacher.skill（教师.skill）

### 产品定位
一个将优秀teacher的教学智慧和人格特质转化为可传承、可迭代AI Skill的工具。

### 目标用户
1. **学校管理者**：留存优秀teacher经验，培训新教师
2. **教师本人**：个人教学风格数字化，专业发展记录
3. **教育研究者**：教师专业发展研究，教学模式分析
4. **师范院校**：优秀教学案例库，师范生培训资源

### 核心价值
1. **知识留存**：防止优秀教学经验因教师离职而流失
2. **经验传承**：新教师可向"数字导师"学习
3. **专业发展**：帮助教师反思、提炼、系统化教学经验
4. **教育公平**：让优质教学资源可复制、可传播

## 问题与机会

### 当前问题
1. **经验流失**：优秀教师退休/离职，经验无法完整传承
2. **培训低效**：新教师成长缓慢，缺乏个性化指导
3. **资源分散**：教案、课件、反思等材料分散，未系统整合
4. **风格难传**：教学风格、课堂艺术等隐性知识难以传递

### 市场机会
1. **教育信息化**：国家推动教育数字化转型
2. **教师专业化**：教师专业标准提升，需要系统支持
3. **AI+教育**：AI技术在教育领域应用深化
4. **终身学习**：教师需要持续专业发展支持

## 功能需求

### 核心功能
#### F1：教师信息采集
- 支持多种数据源：教案、课件、课堂录像、教学反思、学生评价
- 自动解析教学材料，提取结构化信息
- 支持手动补充描述和标签

#### F2：教学能力分析
- 自动分析学科知识体系
- 识别教学方法库和课堂模式
- 提取备课流程和评价策略
- 分析资源建设特点

#### F3：教师风格建模
- 构建5层人格结构模型
- 分析教育理念、沟通风格、决策模式
- 识别师生关系特点
- 生成个性化教师画像

#### F4：Skill生成与管理
- 自动生成教学能力文档（teaching.md）
- 自动生成教师风格文档（teacher.md）
- 生成完整SKILL.md文件
- 版本管理和迭代更新

#### F5：Skill调用与交互
- 支持完整Skill调用（/{slug}）
- 支持仅教学能力调用（/{slug}-teaching）
- 支持仅教师风格调用（/{slug}-teacher）
- 自然语言交互，模拟教师回应

### 扩展功能
#### F6：数据采集扩展
- 飞书/钉钉教学群聊记录采集
- 课堂录像自动转录分析
- 学生作业智能批改记录分析
- 教研活动记录解析

#### F7：分析深度扩展
- 教学效果关联分析
- 专业发展阶段识别
- 教学风格演变追踪
- 跨学科能力迁移分析

#### F8：输出格式扩展
- OpenAI Assistant格式导出
- LangChain Agent格式导出
- 教学案例库格式导出
- 教师培训课程生成

#### F9：协作功能
- 多教师对比分析
- 教研组共同建设
- 师徒结对数字支持
- 校际经验交流

## 用户场景

### 场景1：优秀教师经验留存
**用户**：学校教学副校长
**目标**：留存即将退休的特级教师经验
**流程**：
1. 收集该教师所有教学材料
2. 使用teacher.skill进行分析
3. 生成数字教师Skill
4. 用于新教师培训和教学研究

### 场景2：新教师快速成长
**用户**：入职3年的青年教师
**目标**：向优秀教师学习，快速提升
**流程**：
1. 调用优秀教师Skill（/{slug}-teaching）
2. 咨询备课建议、课堂管理策略
3. 获得个性化指导
4. 记录自身成长，形成个人Skill

### 场景3：教学风格研究
**用户**：教育研究博士生
**目标**：分析不同教师的教学风格差异
**流程**：
1. 采集多位教师材料
2. 批量生成教师Skill
3. 对比分析教学风格
4. 撰写研究论文

### 场景4：校本教研活动
**用户**：教研组长
**目标**：组织有效的教研活动
**流程**：
1. 采集组内教师公开课材料
2. 生成各位教师Skill
3. 对比分析，发现亮点
4. 组织研讨，共同提升

### 技术栈
- **后端**：Python 3.9+
- **前端**：React/Vue（Web界面）
- **AI模型**：OpenAI API / Claude API / 本地LLM
- **文档解析**：python-docx, pdfplumber, python-pptx
- **数据存储**：SQLite / PostgreSQL
- **部署**：Docker, FastAPI

### 数据流程
```
原始材料 → 解析清洗 → 特征提取 → 分析建模 → Skill生成 → 部署调用
    ↓          ↓          ↓          ↓          ↓          ↓
教案/课件  结构化数据  教学特征  教师画像  文档文件  AI交互
```

## 非功能需求

### 性能需求
- **响应时间**：单个教师分析 < 10分钟
- **并发能力**：支持10个并发分析任务

- **可用性**：99%正常运行时间

- **数据加密**：传输和存储加密
- **审计日志**：所有操作可追溯

### 易用性需求
- **学习成本**：30分钟内上手基本功能
- **操作流程**：向导式操作，步骤清晰

### 兼容性需求
- **文件格式**：支持主流办公文档格式

## 项目愿景

- teacher.skill 的最终目标，不是成为一个商业产品，而是一张 **“AI名师资源共享网络”**。

- 一个普通院校的学生，可能因为课程冲突或地域限制，永远无法听到顶尖名师的讲授。而通过本项目的开放架构，一位名师授权公开的资料，就可以化作一张可供所有接入高校学生随时请教的“AI名师卡”。

- 我们相信，技术的价值在于普惠。teacher.skill 希望以极低的成本，把“因材施教”和“与大师对话”的可能性，带给每一位渴望学习的年轻人。

**让教育资源的流动，不再受限于学校的围墙。**
